기본 키 (Primary Key)
1) 테이블에서 각 행을 유일하게 식별하는 컬럼 (식별자)
2) UNIQUE + NOT NULL
3) 보통 단일 컬럼으로 설정합니다. 그러나 여러 컬럼을 조합해서 만들기 가능 (이런 경우 복합 키가 됩니다.)
4) 테이블 당 하나의 기본 키만 존재 가능합니다.

-- student_id가 기본 키: NULL 불가, 중복 불가
CREATE TABLE students (
    student_id INT PRIMARY_KEY,
    name VARCHAR(200)
);

복합 키(Composite Key)
1) 두 개 이상의 컬럼을 조합해서 하나의 기본 키 역할을 하는 키
2) 개별 컬럼은 중복될 수 있어도, 조합된 값이 중복되면 안됩니다.
3) 기본 키를 사용해야 하는데, 단일 컬럼만으로 구분할 수 없을 때

예시)
-- student_id + course_id가 기본 키 역할
-- 같은 학생이 같은 강좌를 두 번 등록할 수 없다.
    CREATE TABLE enrollments (
        student_id INT,
        course_id INT,
        PRIMARY KEY (student_id, course_id)
    );

-- user_id + book_id가 기본 키 역할
-- 한 사람이 같은 책을 두 번 빌릴 수 없다.
    CREATE TABLE borrow_books (
    user_id INT,
    book_id INT,
    PRIMARY KEY (user_id, book_id)
    );
