SQL

1. 개요
    - 명령어는 여러개의 요소로 구성.
    - 이런 요소들을 이해하고, 문법을 익히는 것이 중요합니다.

2. 기본 형식
    - SELECT name, salary FROM employees WHERE salary > 8000;
    - SQL 문장은 보통 세미콜론으로 끝납니다.
    - 반드시 올바른 순서대로 작성이 되어야 한다.
    ex) SELECT -> FROM -> WHERE (올바른 순서)
        FROM -> SELECT (틀린 순서)

3. 주요 구성 요소
    - 키워드(keyword)
        1) SQL에서 특정 작업을 수행하는 명령어
        2) SELECT, FROM, WHERE....등등
        3) 대문자로 쓰는 것이 관례(안써도 되나 그냥 대문자로 쓰자)

    - 식별자(Identifier)
        1) 데이터베이스, 테이블, 컬럼 등의 이름을 나타낸다.
        2) name, salary, employees....

    - 연산자 (Operator)
        1) 데이터를 비교하거나 조작하는 기호
        2) >,<,>=,<=,==,!=

    - 값 (Value)
        1) 구체적인 데이터
        2) 8000

4. 데이터 정의(DDL) vs 데이터 조작(DML)
    -SQL에서 명령어는 크게 DDL과 DML로 구분된다.

    4-1) 데이터 정의(DDL) - Data Definition Language
        1) 데이터 베이스 및 테이블 구조를 정의하는 명령어
        2) 테이블을 만들거나, 수정하거나, 삭제하는 등의 작업 수행

    4-2) 데이터 조작(DML) - Data Manipulation Language
        1) 데이터를 삽입, 조회, 수정, 삭제하는 명령어어
