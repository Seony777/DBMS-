1. 데이터란 무엇일까?
    1) 한마디로, 사실(fact), 기록(record), 관찰(observation)을 통해 얻은 모든 정보

    2) 오늘 내가 아침에 7싱 기상, 아름다운 자연 풍경을 보고 쓴 수필, 학교에서 사람들이 점심 식사하는 걸 구경....모든 것들이 데이터

    3) 아이폰 갤러리의 사진들, 연락처 목록에 저장된 이름, 전화번호, 출석부의 출결 기록... 모든 것들이 데이터

    4) 데이터를 모으는 이유는 데이터를 조합해서 의미를 가지게 하고 싶기 때문
    --> 정보

2. 데이터베이스(Database)란 무엇일까?
    1) 데이터가 너무 많아지면, 효과적으로 저장하고, 관리하고, 검색할 수 있게 만드는 체계적인 저장소가 필요하다. --> 데이터베이스

    2) 데이터베이스는 '데이터의 집'인데, 정해진 구조와 규칙에 따라 관리되는 집

    3) 도서관 예시
        3-1) 도서관에 수많은 책(데이터)이 있다. 이 책들을 특정한 규칙(주제, 분류, 코드 등)에 따라 분류를 한다.
        3-2) 도서관의 책장, 분류 시스템, 관리방식을 모두 포함한 구조가 데이터베이스

    4) 효과
        4-1) 원하는 정보를 빠르게 찾거나 편집 가능
        4-2) 데이터를 안전하게 보관 가능, 여러 사용자가 동시에 사용 가능
        4-3) 데이터가 늘어나도 일관성 있게 유지 가능

3. DBMS(Database Management System)
    1) 데이터베이스를 관리하는 소프트웨어

    2) 도서관 비유
        2-1) 책: 데이터
        책장, 분류 시스템, 관리방식: 데이터 베이스
        책 찾는 걸 도와주기, 책 빌려주기, 반납받기, 새로 들어온 책 분류...
        이런 일을 하는 사람(사서): DBMS

    3) 하는 일
        3-1) 데이터 구조 정의
        3-2) 데이터 추가, 삭제, 수정
        3-3) 데이터 검색, 조회
        3-4) 동시성 제어: 여러 사용자가 동시에 DB를 이용해도 충돌없도록 함
        3-5) 보안 및 권한 관리: 누가 어떤 데이터에 접근하는지, 권한 부여/해제
        3-6) 백업 및 복구: 데이터가 손실될 위험을 대비해서, 백업을 해두고 필요하다면 복구할 수 있도록 기능 제공

4. DBMS의 종류 (RDBMS(SQL) vs NoSQL)
    1) RDBMS(관계형 DBMS): 
    데이터를 테이블 형태로 정형화해서 관리.
    SQL이라는 언어로 데이터를 조회, 조작(Oracle, MySQL, MariaDB, PostgresSQL, MSSQL)

    2) NoSQL:
    비정형 데이터를 유연하게 저장해서 관리.
    (MongoDB, Cassandra, Redis)

    초보 단계일때는, RDBMS를 먼저 배우는게 일반적.
    Oracle, MySQL, MariaDB를 주로 배운다..

5. 오라클 데이터베이스 (Oracle Database) 소개
    5-1) 대기업, 금융권, 공공기관 등 규모가 크고 안정성을 중요시하는 환경에서 많이 쓰이는 DBMS입니다.
    5-2) 다양한 기업 환경에 쓰이는 기능 제공(기술 지원, 고급 기능)
    5-3) 그치만 비싸다.. 현실적으로 비용을 감당하기 쉽지 않다.
    5-4) 무겁고, 설치가 어렵다.

6. MySQL 소개
    6-1) 2008년 Sun Microsystems에서 개발 -> 2010년 Oracle이 Sun을 인수
    6-2) 오픈소스(커뮤니티 버전-무료버전), 기업용(엔터프라이즈 버전)은 별도 판매
    6-3) 커뮤니티 버전도 성능이 충분히 좋습니다. 스타트업, 중소기업, 중견기업.. 많이 사용합니다.
    6-4) 가볍고, 빠르고, 설치가 쉽다. -->  초보자가 선택하기 매우 좋다.

7. MariaDB 소개 (MySQL과 거의 비슷)
    7-1) MySQL 창시자가 오픈소스 정신을 유지하기 위해 만든 프로젝트
    7-2) MySQL과 매우 유사하며, 내부 엔진, 설정, SQL문법 호환성이 엄청 높다. --> MySQL과 거의 동일한 방식으로 사용가능
    7-3) 오픈소스이기 때문에 무료. 기업용 지원 패키지는 유료로 판매

8. MySQL의 장점
    8-1) 진입 장벽이 낮다. 커뮤니티 버전이 강력하다.
    8-2) MariaDB도 좋은 선택이지만, MySQL관련 자료, 튜토리얼, 학습 사이트 등등 훨씬 풍부
    8-3) DB-engines ranking -> 1등 Oracle / 2등 MySQL
    8-4) 실무에서 많이 사용되고, 그만큼 수요가 높다.
