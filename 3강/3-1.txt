Timezone 처리
1) MySQL의 TIMESTAMP 내부적으로 UTC(국제표준준) 기준으로 저장한 뒤,
클라이언트 / 서버 시간대에 맞춰 변환한다.

2) 웹, 모바일 어플리케이션 서버나 MySQL 설정에 따라 시차가 달라질 수 있으니, UTC기준을 사용하거나 혹은 규칙을 정하는게 좋다.

ENUM / BOOLEAN
1) ENUM은 값 제한이 필요하고, 간단한 경우에만 사용하는 것이 좋습니다.
2) BOOLEAN은 내부적으로 TINYINT(1) 취급되므로, TRUE/FALSE 외에도 0/1로도 입력가능하다.

SQL 파일 구조
1) 규모가 커지면, DDL(테이블 생성)과 DML(데이터 삽입/조회)뿐만 아니라, ALTER(구조 변경), DROP(삭제) 등을 SQL 파일로 구분해 관리해야 합니다.
2) 협업 시에 버전 관리(Git, Github)를 하면서, SQL 파일을 공유하는 방식을 사용한다.
