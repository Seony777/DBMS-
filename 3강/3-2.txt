1. 기본값(DEFAULT) 지정하기
예시: orders 테이블에서 order_date라는 컬럼이 존재한다
    1) 매번 주문 시간이 들어갈 때마다, 현재 시각을 직접 입력해줄 수 없다. 또 실수가 생길 수 있다.

    2) 이런 경우, 기본값을 현재 시각(CURRENT_TIMESTAMP)으로 설정하면, 데이터를 삽입할 때, 값을 생략해도 된다. 값을 생략하면, 자동으로 현재 시간을 기록해준다.

    3) 테이블 생성 시 DEFAULT 지정
    CREATE TABLE IF NOT EXISTS orders(
        order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        quantity INT,
        order_message TEXT 
    );

    4) 이미 만들어진 테이블에 DEFAULT 추가하기
    ALTER TABLE orders
    MODIFY CoLUMN oreder_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

2. 테이블 삭제(DROP)와 재생성
    1) DROP TABLE
    테이블 구조 자체를 지우고, 다시 만드는 경우에 사용한다.
    이미 존재하는 테이블에 데이터가 있지만, 전부 삭제해도 상관없다면 
    이 방법을 사용하세요.(연습 또는 테스트 환경에서 많이 씀씀)

    DROP TABLE IF EXISTS customers;
    DROP TABLE customers;

    2) CREATE TABLE
    재생성을 하고 싶을 때도 사용 가능합니다.
    기존 데이터가 필요하다면, DROP보다는 ALTER

3. NULL vs NOT NULL 제약
    1) NULL과 0(또는 빈 문자열)의 차이이 
        NULL은 '값이 존재하지 않음'을 의미한다. 특별한 값
        
        0,''(빈 문자열)은 값은 존재하지만, 그 값이 0이거나 빈 문자열이라는 의미

        그래서, PRICE=0과 PRICE = NULL 전혀 다른 의미
        0은 말 그대로 가격이 0원.
        NULL은 가격 정보가 입력되지 않음

    2) NULL을 활용하는 이유
        예를 들어서, customers 테이블에서 어떤 고객의 email이 입력되지 않은 경우(모르는 경우, 선택사항의 경우) - NULL 저장 가능
        통계나 평균을 계산할 때, NULL 값은 계산에서 제외된다.

    3) NOT NULL로 값 입력 강제(필수 입력 사항들)
        일부 컬럼에 항상 값이 존재해야 한다면 (title, price 등등) NOT NULL 제약을 추가할 수 있습니다.
        
        CREATE TABLE IF NOT EXISTS books (
            title VARCHAR(200) NOT NULL, -- 책 제목은 항상 필요 
            author VARCHAR(100),
            price DECIMAL(6,2),
            is_published BOOLEAN
        );
        ALTER TABLE books
        MODIFY COLUMN title VARCHAR(200) NOT NULL;

4. NULL 데이터 삽입 방법
    1) 컬럼 생략
    INSERT 시, 특정 컬럼을 아예 빼면, 그 컬럼은 자동으로 NULL이 들어간다. (단, NOT NULL이 아니어야 하겠죠)
    -- order_date 컬럼을 제외하고 값을 삽입하고 있음음
    INSERT INTO orders(quantity,order_message)
    VALUES  
    (4,'항상 감사합니다.');

    2) DEFAULT와 NULL의 관계
    어떤 컬럼에 DEFAULT 값이 저장되어 있으면, INSERT시 해당 컬럼을 삭제했을 때, NULL 대신 DEFAULT 값이 들어간다.
    DEFAULT가 적용된 컬럼에 명시적으로 NULL을 넣고 싶다면, NULL을 표기해줘야 한다.

    3) 대부분의 경우에는 DEFAULT로 지정된 컬럼은 기본값이 들어가기 때문에, NULL에 관련된 문제는 생기지 않는다.
